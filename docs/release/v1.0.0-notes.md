## Overview

Lan is a fast, skill-native TUI agent written in Zig. It provides an interactive terminal interface for AI-assisted development with support for multiple LLM providers, tool calling, and an extensible skill system.

## What's New in v1.0.0

### Core TUI Experience
- **Streaming responses** - See AI output as it's generated
- **Beautiful box-drawing UI** - Clean, organized terminal interface
- **Syntax highlighting** - Code blocks rendered with colors
- **Markdown support** - Bold, italic, inline code formatting
- **Multiline input** - Full editing capabilities
- **Command mode** - `/help`, `/clear`, `/exit` shortcuts

### Multi-Provider Support
- **Kimi** (Moonshot) - Default provider
- **Anthropic** (Claude) - Full support
- **OpenAI** (GPT) - Full support
- **Automatic fallback** - Switch providers on failure with logging

### Tool Calling
- `read_file` - Read file contents
- `write_file` - Write/modify files
- `list_directory` - List directory contents
- `exec` - Execute shell commands (with caution)

### Skill System
- `lan skill add <path>` - Install local skills
- `lan skill remove <name>` - Remove installed skills
- `lan skill list` - List all skills
- `lan skill info <name>` - Show skill details
- `lan skill search <keyword>` - Search skill registry (stub)
- **Manifest-based** - JSON configuration for permissions and tools

### Session Management
- `lan history export` - Export as JSON or Markdown
- `lan history search <query>` - Search history
- `lan history clear` - Clear history
- `lan history stats` - Show session statistics
- **Auto-save** - History persisted automatically

### Configuration
- `lan config init` - Create default config
- `lan config reload` - Hot-reload config changes
- **Environment variables** - `MOONSHOT_API_KEY`, `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`
- **Config file** - `~/.config/lan/config.json`

### Diagnostics
- `lan doctor` - Comprehensive system diagnostics
- `lan --version` / `lan version` - Version information
- **Support bundle** - `scripts/support-bundle.sh` for troubleshooting

## Installation

### From Release (Recommended)

```bash
# Download
curl -fsSL https://github.com/ekil1100/lan/releases/download/v1.0.0/lan-$(uname -s | tr '[:upper:]' '[:lower:]')-$(uname -m).tar.gz -o lan.tar.gz

# Install
./scripts/install.sh lan.tar.gz ~/.local/bin

# Verify
lan --version
```

### From Source

```bash
git clone https://github.com/ekil1100/lan.git
cd lan
zig build
./zig-out/bin/lan
```

## Known Limitations

- **Platform support**: macOS arm64 and Linux x86_64 only
- **API keys**: Required for online chat (bring your own)
- **Skill registry**: Remote skill installation pending registry hosting
- **Windows**: Not yet supported

## Upgrade Guide

### From v0.1.0-beta

```bash
# Backup existing config
cp ~/.config/lan/config.json ~/.config/lan/config.json.bak

# Download and install new version
curl -fsSL https://github.com/ekil1100/lan/releases/download/v1.0.0/lan-$(uname -s | tr '[:upper:]' '[:lower:]')-$(uname -m).tar.gz -o lan.tar.gz
./scripts/install.sh lan.tar.gz ~/.local/bin

# Config is compatible, no migration needed
lan --version  # Should show v1.0.0
```

## Error Codes

Lan uses standardized error codes (E1xx-E5xx):
- **E1xx**: Configuration errors
- **E2xx**: Session/history errors  
- **E3xx**: Install/upgrade errors
- **E4xx**: Skill management errors
- **E5xx**: CI/CD errors

See `docs/errors.md` for complete reference.
